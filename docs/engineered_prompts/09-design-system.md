SYSTEM: Chronos Sentinel – Design System (Stage 1)

OBJECTIVE:
Ensure a clean, consistent, intelligence-first UI.
Avoid visual noise. Prioritize clarity, speed, and trust.

---

SECTION 1 — DESIGN PRINCIPLE

This is NOT a media site.

This is:
- an intelligence interface
- a signal extraction tool

RULE:
Text-first. No unnecessary images.

---

SECTION 2 — VISUAL STYLE

Tone:
- minimal
- serious
- analytical
- non-entertainment

Avoid:
- bright colors
- flashy elements
- content thumbnails
- visual clutter

---

SECTION 3 — COLOR SYSTEM

Use color ONLY for meaning.

GREEN:
- normal / low-risk / stable

YELLOW:
- moderate significance

ORANGE:
- high activity / attention needed

RED:
- critical / urgent

NEVER use color decoratively.

---

SECTION 4 — TYPOGRAPHY HIERARCHY

Level 1 (Primary): Section titles - bold, clear
Level 2 (Card titles): cluster titles
Level 3 (Body): summaries, facts
Level 4 (Meta): timestamps, sources

RULE:
Clear hierarchy. No mixed emphasis.

---

SECTION 5 — CARD STRUCTURE (MANDATORY)

Each synthesis card MUST include:
1. Title (event/topic)
2. Timestamp (Year-Month-Day HH:mm)
3. Shared Facts
4. Source Differences
5. Synthesis Summary
6. Confidence
7. Sources

DO NOT remove sections.

---

SECTION 6 — TERMINOLOGY (STRICT)

Use: "Sources", "Synthesis", "Shared Facts", "Differences", "Confidence".

---

SECTION 7 — TIMESTAMP FORMAT

All time must include date.
Format: YYYY-MM-DD HH:mm

---

SECTION 8 — NAVIGATION RULES

Mobile: bottom navigation allowed
Desktop: MUST be top sticky navigation

---
